商品管理システム要件定義書

1. はじめに

1.1 目的

本文書は、商品管理システムの開発に関する要件を定義するものです。商品の登録、更新、検索、および在庫管理を効率的に行うためのシステム要件を明確にします。

1.2 対象範囲

本システムは以下の機能を対象とします：

  • 商品データの登録・更新・削除
  • 商品カテゴリ管理
  • 在庫管理
  • 商品検索
  • レポート生成
  • ユーザー管理

2. システム概要

2.1 システム構成

本システムはWebアプリケーションとして実装され、以下のコンポーネントで構成されます：

  • フロントエンド：HTML, CSS, JavaScript (React.js)
  • バックエンド：Node.js
  • データベース：MySQL

2.2 利用者

本システムの利用者は以下の通りです：

  • 管理者：システム全体の管理権限を持ちます
  • 商品管理者：商品情報の登録・更新権限を持ちます
  • 在庫管理者：在庫状況の確認・更新権限を持ちます
  • 閲覧者：情報の閲覧のみ可能です

3. 機能要件

3.1 商品管理機能

ID 機能名 説明 優先度
F-001 商品登録 新規商品情報を登録する。商品名、商品コード、カテゴリ、価格、説明、画像等の基本情報を入力可能。 高
F-002 商品情報更新 既存の商品情報を更新する。 高
F-003 商品情報削除 商品情報を論理削除する（完全に削除せず、非表示フラグを立てる）。 中
F-004 商品情報一覧表示 登録されている商品情報を一覧で表示する。ページング機能付き。 高
F-005 商品詳細表示 商品の詳細情報を表示する。 高

3.2 カテゴリ管理機能

ID 機能名 説明 優先度
F-006 カテゴリ登録 新規カテゴリを登録する。 高
F-007 カテゴリ更新 既存のカテゴリを更新する。 中
F-008 カテゴリ削除 カテゴリを削除する（関連商品がある場合は削除不可）。 中
F-009 カテゴリ一覧表示 カテゴリを一覧で表示する。 高

3.3 在庫管理機能

ID 機能名 説明 優先度
F-010 在庫登録 商品の初期在庫数を登録する。 高
F-011 在庫更新 商品の在庫数を増減させる。 高
F-012 在庫一覧表示 全商品の在庫状況を一覧で表示する。 高
F-013 在庫アラート 在庫が設定値を下回った際にアラートを表示する。 中

3.4 検索機能

ID 機能名 説明 優先度
F-014 商品検索 キーワードによる商品検索を行う。商品名、コード、説明文から検索可能。 高
F-015 カテゴリ検索 カテゴリによる商品絞り込みを行う。 中
F-016 詳細検索 複数条件（価格帯、登録日、在庫状況など）による詳細検索を行う。 低

3.5 レポート機能

ID 機能名 説明 優先度
F-017 在庫状況レポート 全商品の在庫状況レポートを生成する。CSVエクスポート可能。 中
F-018 入出庫履歴レポート 指定期間の入出庫履歴レポートを生成する。CSVエクスポート可能。 中

3.6 ユーザー管理機能

ID 機能名 説明 優先度
F-019 ユーザー登録 システム利用ユーザーを登録する。 高
F-020 ユーザー更新 ユーザー情報を更新する。 中
F-021 ユーザー削除 ユーザーを削除する（論理削除）。 中
F-022 ユーザー一覧表示 登録ユーザーを一覧で表示する。 中
F-023 ログイン ID・パスワードによるログイン認証を行う。 高
F-024 ログアウト システムからログアウトする。 高
F-025 パスワードリセット パスワードを再設定する。 中

4. 非機能要件

4.1 性能要件

ID 項目 要件 優先度
NF-001 応答時間 一般的な操作の応答時間は3秒以内とする。 高
NF-002 同時接続数 最大50ユーザーの同時接続に対応する。 高
NF-003 データ処理量 最大10万件の商品データを管理可能とする。 中

4.2 セキュリティ要件

ID 項目 要件 優先度
NF-004 認証 ユーザーID・パスワードによる認証を実施する。パスワードは8文字以上、英数字混在とする。 高
NF-005 権限管理 ユーザー種別ごとにアクセス権限を設定し、不正なアクセスを防止する。 高
NF-006 データ保護 通信データはSSL/TLSで暗号化する。パスワードはハッシュ化して保存する。 高

4.3 可用性要件

ID 項目 要件 優先度
NF-007 稼働時間 24時間365日の稼働を基本とし、計画的なメンテナンス時間を除き、99.5%以上の可用性を確保する。 高
NF-008 バックアップ データベースの定期バックアップを実施する（毎日1回）。 高

4.4 保守性要件

ID 項目 要件 優先度
NF-009 拡張性 将来的な機能追加やデータ増加に対応できる拡張性を持たせる。 中
NF-010 保守性 コードの可読性を高め、保守作業が容易になるよう設計する。 中

5. 画面要件

5.1 画面一覧

ID 画面名 説明 優先度
S-001 ログイン画面 システムへのログイン画面。 高
S-002 ダッシュボード ログイン後に表示されるトップ画面。全体の状況を把握できる。 高
S-003 商品一覧画面 商品を一覧表示する画面。検索・ソート機能付き。 高
S-004 商品詳細画面 商品の詳細情報を表示する画面。 高
S-005 商品登録画面 商品を新規登録する画面。 高
S-006 商品編集画面 商品情報を編集する画面。 高
S-007 カテゴリ管理画面 カテゴリの登録・編集・削除を行う画面。 中
S-008 在庫一覧画面 在庫状況を一覧表示する画面。 高
S-009 在庫更新画面 在庫数を更新する画面。入出庫理由も記録可能。 高
S-010 ユーザー管理画面 ユーザーの登録・編集・削除を行う画面。 中
S-011 レポート画面 各種レポートを生成・表示する画面。 中

6. データベース要件

6.1 テーブル構成

以下の主要テーブルを設計します：

テーブル名 説明 主要項目
products 商品マスタテーブル id, code, name, description, price, category_id, image_path, created_at, updated_at, deleted_at
categories カテゴリマスタテーブル id, name, parent_id, created_at, updated_at
inventories 在庫テーブル id, product_id, quantity, alert_threshold, created_at, updated_at
inventory_histories 在庫履歴テーブル id, product_id, quantity_change, reason, operated_by, created_at
users ユーザーテーブル id, username, password_hash, email, role, last_login_at, created_at, updated_at, deleted_at

9. 付録

9.1 用語集

用語 説明
SKU Stock Keeping Unit（在庫管理単位）。商品を識別するためのコード。
入出庫 商品の入庫（在庫増加）と出庫（在庫減少）を表す用語。
論理削除 データベースから実際に削除せず、削除フラグを立てて表示上は削除されたように見せる方式。
